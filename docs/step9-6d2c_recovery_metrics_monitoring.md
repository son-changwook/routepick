# 장애 복구 매트릭스 및 모니터링

## 장애 복구 매트릭스

### 1. 복구 시간 목표 (RTO: Recovery Time Objective)
- **데이터베이스 장애**: 30초 이내
- **캐시 장애**: 10초 이내  
- **외부 API 장애**: 5초 이내
- **메모리 부족**: 20초 이내
- **네트워크 분할**: 60초 이내

### 2. 복구 지점 목표 (RPO: Recovery Point Objective)  
- **사용자 데이터**: 0초 (무손실)
- **메시지 데이터**: 5초 이내
- **결제 데이터**: 0초 (무손실)
- **시스템 로그**: 30초 이내

### 3. 가용성 목표
- **전체 시스템**: 99.9% (월 43분 다운타임)
- **핵심 서비스**: 99.95% (월 22분 다운타임)
- **결제 시스템**: 99.99% (월 4분 다운타임)

### 4. 복구 전략
- **자동 복구**: 80% 장애 시나리오
- **반자동 복구**: 15% 장애 시나리오  
- **수동 복구**: 5% 장애 시나리오
